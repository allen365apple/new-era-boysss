# 新世紀直男戰士 — 網站系統架構與內容流程

> 這份文件說明「新世紀直男戰士」部落格網站的完整系統：它由哪些部分組成、如何運作、文章怎麼從 Podcast 變成網頁。
> 目標是讓**任何人或任何 AI** 看完就能理解並接手維護。

---

## 1. 這個系統在做什麼？

把 Podcast 節目《新世紀直男戰士》的內容，做成一個**部落格網站**，把每集的對談整理成文章發布。

兩大核心目標：

- **SEO**：讓在 Google 搜尋性別議題的讀者能找到文章。
- **AEO**：讓 AI（ChatGPT、Claude、Perplexity 等）回答性別問題時能引用本站內容。

線上網址：`https://new-era-boysss.pages.dev`

---

## 2. 系統全貌（各部分如何串接）

```
   ┌──────────────┐   寫程式/文章    ┌──────────────┐   偵測更新    ┌──────────────────┐
   │  你的電腦      │ ───────────────▶ │   GitHub      │ ────────────▶ │  Cloudflare Pages │
   │ (Astro 原始碼) │                  │ (原始碼+文章)  │   自動部署     │  (編譯+全球上線)   │
   └──────────────┘                  └──────────────┘               └──────────────────┘
                                            ▲                                  │
                          夥伴用 GitHub 登入  │                                  ▼
                       ┌──────────────────┐  │                         讀者看到網站
                       │  Decap CMS 後台    │──┘                      new-era-boysss.pages.dev
                       │  (/admin 寫文章)   │
                       └──────────────────┘
```

用一句比喻：**GitHub 是倉庫，Cloudflare 是自動工廠兼展示間，Astro 是排版師傅。**
你（或夥伴）把稿件放進倉庫，工廠自動把它排版成網頁、推送到全世界。

## 3. 技術組成（全部免費方案）

| 工具 | 角色 | 說明 |
|------|------|------|
| **Astro** (v7) | 前台框架 | 把文章 + 版型「編譯」成純 HTML 靜態網站，SEO 友善、速度快 |
| **Decap CMS** | 後台 | 夥伴用各自的 GitHub 帳號登入 `/admin` 寫文章，全程不碰程式碼 |
| **GitHub** | 原始碼 + 文章倉庫 | 存程式碼與 markdown 文章，記錄所有版本 |
| **Cloudflare Pages** | 架站 + 自動部署 | 連到 GitHub repo，每次 push 自動重新編譯上線；並用 Functions 處理後台登入 |
| **Noto Sans TC** | 字體 | 思源黑體，透過 Google Fonts 載入（CJK 動態子集化） |

- **Repo**：`allen365apple/new-era-boysss`
- **託管平台**：Cloudflare Pages（框架預設 Astro，建置指令 `npm run build`，輸出目錄 `dist`）

## 4. 部署流程（push 後會發生什麼）

1. 程式碼或文章變動 → `git push` 到 GitHub（夥伴在後台 Publish 也會觸發）。
2. Cloudflare Pages 偵測到 GitHub 有新 commit。
3. Cloudflare 自動執行 `npm run build`：Astro 把所有文章套版型，產生純 HTML。
4. 編譯結果部署到 Cloudflare 全球節點，網站更新（通常 20–60 秒）。

> 不需要手動上傳、不需要 GitHub Actions 設定。push 即上線。

## 5. 文章資料結構（frontmatter schema）

每篇文章是一個 markdown 檔，放在 `src/content/blog/`。檔案最上方的 frontmatter 欄位定義在 `src/content.config.ts`：

| 欄位 | 型別 | 說明 |
|------|------|------|
| `title` | 文字 | 文章標題 |
| `description` | 文字 | 摘要，給 SEO meta 用（建議 50–160 字） |
| `pubDate` | 日期 | 發布日期 |
| `updatedDate` | 日期（選填） | 最後更新日 |
| `episode` | 文字（選填） | 對應集數，如 `EP52` |
| `guests` | 文字（選填） | 來賓 |
| `topics` | 陣列（選填） | 議題分類，限以下 7 種 |
| `listenLinks` | 物件（選填） | 收聽連結：`soundon` / `spotify` / `apple` / `kkbox` |
| `coverImage` | 圖片（選填） | 封面圖（本機圖檔，方形最佳） |
| `aiGenerated` | 布林（預設 true） | 是否為 AI 整理；true 時文末顯示 AI 說明 |
| `draft` | 布林（預設 false） | true 則不發布（草稿） |

**議題分類（topics）固定七種**，與節目企劃一致：
`男性特權`、`情感腳本`、`性別凝視`、`身份認同`、`在地事件`、`多元對話`、`聽眾互動`

## 6. 設計系統（改一處、全站連動）

所有顏色、字體、間距都定義在 `src/styles/global.css` 最上方的 `:root` 區塊（design tokens）。改一個變數，全站對應的地方一起變。

目前風格參照 RicoFast：暖米色底、皇家藍標題、虛線分隔線。詳細的「改風格教學」見 **`客製化指南.md`**。

## 7. CMS 後台（夥伴怎麼發文）

- 後台網址：`/admin`（檔案在 `public/admin/index.html` 與 `config.yml`）。
- **登入模型**：每位夥伴用**自己的 GitHub 帳號**登入，不共用帳號。需先被加入 repo 的 collaborator。
- **登入驗證**：GitHub OAuth 握手由 Cloudflare Pages Functions 處理（`functions/api/auth.js`、`functions/api/callback.js`），金鑰存在 Cloudflare 環境變數 `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET`。
- 夥伴日常流程：打開 `/admin` → GitHub 登入 → 寫文/改文 → Publish → 自動上線，全程不碰 git。

## 8. SEO / AEO 體質

- **sitemap**：`@astrojs/sitemap` 自動產生 `/sitemap-index.xml`。
- **robots.txt**（`public/robots.txt`）：**明確允許 AI 爬蟲**（GPTBot、ClaudeBot、PerplexityBot、Google-Extended 等）。
- **每頁 meta**：title、description、canonical、Open Graph、Twitter Card（共用元件 `BaseHead.astro`）。
- **結構化資料 JSON-LD**：文章頁嵌入 `BlogPosting`，首頁/關於頁嵌入 `PodcastSeries`。
- **語意化 HTML**：每頁一個 h1、內文 h2/h3 分層，方便 AI 擷取重點。
- 文章頁左側有**自動目錄 + 捲動高亮**，由 markdown 標題自動生成。

> AEO（被 AI 引用）是「提高機率」的最佳實作，非保證；領域變動快，日後可再調整。

## 9. 內容生產線：Podcast → 逐字稿 → 文章

這是把每集節目變成文章的流程。**目前採「半自動 + 人工審稿」**，品質由人把關。

```
① 新節目上架        節目發布在 SoundOn，RSS 自動推送到各平台
② 取得逐字稿        用 Memo AI（永久會員，免費）轉錄該集 → 匯出 Markdown
③ 寫文章            把逐字稿交給 Claude，依既定寫法產出文章草稿
④ 進後台草稿        文章以 draft: true 進到 /admin
⑤ 人工審稿          製作團隊在後台審閱、修改
⑥ 發布上線          按 Publish → Cloudflare 自動部署
```

### 來源 RSS
- SoundOn 音檔 RSS：`https://feeds.soundon.fm/podcasts/77af8789-ee40-4dd4-94b9-35c32ed1a46a.xml`
- YouTube RSS：`https://feeds.soundon.fm/podcasts/77af8789-ee40-4dd4-94b9-35c32ed1a46a/yt.xml`

### 為什麼是「半自動」而非「全自動」
- **轉逐字稿用 Memo AI**：品質好、已付費（永久會員）、本機處理免費。但 Memo AI 是**桌面 App，沒有公開 API**，無法被機器人自動呼叫——所以「轉錄」這步一定要人工操作（開 App、貼連結、匯出）。
- **這正好符合品質策略**：性別議題文章品質重要，且系統設計成「自動產草稿、人工審核後才發布」，保留完全的品質控制。

### 成本
- Memo AI 轉錄：**免費**（已是永久會員）。
- Claude 寫文章：可在 Claude Code 對話中進行（不另計費），或改用 Claude API（每篇約 US$0.1–0.2）。
- Cloudflare Pages / GitHub：**免費**。

### 未來若要更自動化
若改用有 API 的轉錄服務（如 OpenAI Whisper，約 US$0.4–0.6/集），可用 **GitHub Actions 排程**（public repo 免費）做到「定時檢查 RSS → 下載音檔 → 自動轉錄 → Claude 寫稿 → commit 草稿」的近乎全自動。RSS 本身不能「推播」，只能定時檢查。

## 10. 檔案結構

```
site/
├── astro.config.mjs          # Astro 設定（site 網址、sitemap、mdx）
├── src/
│   ├── consts.ts             # 網站標題、描述等品牌資訊
│   ├── content.config.ts     # 文章 frontmatter schema（第 5 節）
│   ├── content/blog/         # 所有文章（.md / .mdx）
│   ├── pages/
│   │   ├── index.astro       # 首頁（文章列表）
│   │   ├── about.astro       # 關於節目
│   │   └── blog/[...slug].astro  # 單篇文章路由
│   ├── layouts/BlogPost.astro    # 文章版型（目錄、動畫、收聽區塊）
│   ├── components/
│   │   ├── BaseHead.astro    # SEO meta + JSON-LD
│   │   ├── Header.astro / Footer.astro
│   │   └── FormattedDate.astro / HeaderLink.astro
│   ├── styles/global.css     # 設計變數（design tokens）
│   └── assets/               # 圖片、字體
├── public/
│   ├── robots.txt            # 允許 AI 爬蟲
│   └── admin/                # Decap CMS 後台（index.html + config.yml）
├── functions/api/            # Cloudflare OAuth 握手（auth.js, callback.js）
├── 客製化指南.md              # 改風格教學（顏色/字體/版面）
└── 系統架構.md                # 本文件
```

## 11. 日常操作速查

| 我想要… | 怎麼做 |
|---------|--------|
| 發一篇新文章 | 打開 `/admin` → GitHub 登入 → 新增內容 → 填欄位 → Publish |
| 把節目轉成文章 | Memo AI 轉逐字稿 → 交給 Claude 寫草稿 → 後台審核 → 發布 |
| 改網站風格（顏色/字體） | 改 `src/styles/global.css` 的 `:root` 變數（見 `客製化指南.md`） |
| 改節目介紹 | 改 `src/pages/about.astro` |
| 新增夥伴 | GitHub repo → Settings → Collaborators → 邀請 |
| 在本機預覽 | `npm run dev` → 開 `localhost:4321` |

---

## 12. 重要原則

- **文章品質優先**：AI 只產草稿，最終一定經人工審稿才發布。
- **改風格集中管理**：只改 `global.css` 的 `:root`，不要在各元件裡零散改色。
- **公開前檢查個資**：本網站與 repo 為公開，切勿放入成員真實全名、聯絡電話、Email、地址等敏感資訊。
